# Mindeleone: функциональное описание для веб-реализации

## 1. Назначение

Программа представляет собой интерактивный редактор карты мыслей: пользователь создает узлы, связывает их линиями, перемещает по бесконечной рабочей области, меняет размер и цвет, просматривает связи выбранного узла, сохраняет карту в текстовый формат и восстанавливает ее из этого формата.

## 2. Основные сущности данных

### 2.1. Узел

Каждый узел хранит и отображает следующие данные:

* уникальный строковый идентификатор;
* заголовок;
* координаты на рабочей области;
* размер;
* цвет в RGB-формате с компонентами от 0 до 1;
* статус;
* список связей с другими узлами.

Размер узла ограничен диапазоном от 1 до 100. По умолчанию новый узел имеет статус «актуальный», размер 1 и заголовок, переданный при создании.

### 2.2. Статусы узла

Поддерживаются три статуса:

* `ACTUAL` — актуальный;
* `COMPLETED` — завершенный;
* `IRRELEVANT` — неактуальный.

В текущем поведении при загрузке из сохранения фактически восстанавливаются только актуальные узлы; узлы с другим статусом пропускаются.

### 2.3. Связь

Связь соединяет два узла и визуализируется линией между их текущими координатами. Связь хранится симметрично: у каждого из двух узлов есть собственная запись связи на противоположный узел. При удалении связи или узла соответствующая встречная связь также удаляется.

Цвет линии должен начинаться цветом текущего узла и заканчиваться цветом связанного узла. При изменении цвета узла цвета всех связанных с ним линий обновляются.

## 3. Рабочая область и камера

Пользователь работает с двумерной рабочей областью, на которой размещены узлы и линии связей.

### 3.1. Перемещение рабочей области

* При удержании левой кнопки мыши и перемещении курсора камера/вид сдвигается в направлении, соответствующем разнице между предыдущей и текущей позицией курсора.
* Если пользователь начинает перетаскивать узел, перемещение камеры в этот момент отключается, чтобы узел двигался вместо рабочей области.

### 3.2. Масштабирование

* Колесо мыши изменяет масштаб вида.
* Масштабирование не выполняется, если одновременно удерживается `Shift`.
* Скорость масштабирования пропорциональна текущему масштабу: чем больше масштаб, тем крупнее шаг изменения.

### 3.3. Фокусировка на выбранном узле

Комбинация `Ctrl + F` перемещает вид в координаты выбранного узла и устанавливает масштаб равным удвоенному размеру этого узла.

## 4. Создание, выбор, редактирование и удаление узлов

### 4.1. Начальное состояние

После запуска выбирается первый узел из исходного списка узлов. При старте также выполняется автозагрузка сохраненной карты из локального хранилища приложения, если сохранение есть.

### 4.2. Создание нового узла

Новый узел можно создать двумя способами:

* нажатием клавиши `Tab`;
* нажатием кнопки «Spawn node!».

При создании нового узла:

1. перед созданием сохраняется текущий текст выбранного узла из поля редактирования;
2. узел создается рядом с текущим выбранным узлом: на одну условную единицу ниже, с небольшим случайным смещением по обеим осям;
3. если при создании был выбран исходный узел, новый узел автоматически связывается с ним;
4. новый узел добавляется в общий список узлов;
5. новый узел становится выбранным;
6. поле редактирования заголовка переводится в режим ввода, курсор ставится в конец текста.

### 4.3. Выбор узла

При клике по узлу без управляющих клавиш он становится выбранным. При выборе:

* ранее выбранный узел сохраняет введенный в поле заголовок и снимает визуальное выделение;
* новый выбранный узел получает визуальное выделение;
* поле редактирования заголовка заполняется заголовком выбранного узла;
* панель связей перестраивается под выбранный узел.

### 4.4. Редактирование заголовка

Заголовок выбранного узла редактируется через отдельное текстовое поле интерфейса.

* Нажатие `Enter` сохраняет текст поля в выбранный узел.
* При переключении выбора или создании нового узла текст также сохраняется автоматически.
* Заголовок обновляется только если после очистки ввода длина текста больше 1 символа.
* При изменении заголовка обновляется отображаемый текст узла и внутреннее имя узла.

### 4.5. Валидация ввода

Поле заголовка фильтрует ввод и разрешает только следующие символы:

* латинские буквы;
* кириллические буквы;
* цифры;
* пробел;
* символы: `` ` ! @ # $ % ^ & * ( ) _ + - = \ { } [ ] : " ' < > ? , . / ``.

Все остальные символы автоматически удаляются из поля.

### 4.6. Перетаскивание узлов

Узел можно перетаскивать мышью. При начале перетаскивания запоминается смещение между центром узла и позицией курсора, поэтому узел не «прыгает» центром под курсор. Во время перетаскивания координаты узла обновляются, а связующие линии автоматически следуют за ним.

### 4.7. Удаление узла

Выбранный узел удаляется при нажатии:

* `Delete`;
* или `Right Command + Backspace`.

При удалении узла удаляются все его связи, соответствующие встречные связи у соседних узлов и сам узел исключается из общего списка.

## 5. Связи между узлами

### 5.1. Создание связи

Чтобы создать связь:

1. выбрать первый узел;
2. удерживать `Left Shift`;
3. кликнуть по другому узлу.

Если связь между этими узлами уже существует, новая связь не создается, а используется существующая.

После создания связи размер узла, на котором создавалась связь, становится равен количеству его связей. Затем изменение размера распространяется на соседей с затухающим коэффициентом.

### 5.2. Удаление связи

Чтобы удалить связь:

1. выбрать один из связанных узлов;
2. удерживать `Left Alt`;
3. кликнуть по другому связанному узлу.

Удаление выполняется симметрично: связь удаляется у обоих узлов, а линия исчезает.

### 5.3. Автообновление линии

Каждая линия на каждом кадре обновляет свои начальную и конечную точки по текущим координатам связанных узлов. Поэтому линии корректно отображаются при перемещении узлов.

### 5.4. Видимость связей

При скрытии узла скрываются линии, ведущие к нему. При показе узла эти линии снова отображаются.

## 6. Размер узлов

### 6.1. Ручное изменение размера

Если выбранный узел находится под курсором, пользователь удерживает `Left Shift` и прокручивает колесо мыши, размер узла изменяется на величину прокрутки.

### 6.2. Ограничение размера

Размер узла всегда ограничивается диапазоном от 1 до 100.

### 6.3. Распространение изменения размера на соседей

При изменении размера узла изменение рекурсивно распространяется на связанные узлы:

* каждый следующий сосед получает половину изменения предыдущего уровня;
* распространение прекращается, когда абсолютная величина изменения становится меньше 0,1;
* для предотвращения бесконечных циклов каждый узел в рамках одного распространения обрабатывается не более одного раза.

## 7. Цвет узлов и линий

Для выбранного узла доступны три горизонтальных регулятора цвета: красный, зеленый и синий каналы. Каждый канал принимает значение от 0 до 1.

При изменении цвета выбранного узла:

* меняется цвет самого узла;
* обновляются цвета всех линий, связанных с этим узлом;
* у каждой линии начало окрашивается в цвет текущего узла, конец — в цвет связанного узла.

## 8. Панель связей выбранного узла

Для выбранного узла показывается список связанных с ним узлов.

Поведение панели:

* перед обновлением список полностью очищается;
* связи сортируются по размеру связанных узлов по убыванию;
* для каждой связи создается кнопка;
* текст кнопки имеет формат: `(<размер>) <заголовок связанного узла>`;
* цвет кнопки основан на цвете связанного узла, но с уменьшенной прозрачностью;
* нажатие кнопки выбирает связанный узел и фокусирует на нем вид.

## 9. Фильтрация видимости узлов

В программе есть список статусов, которые должны быть видимыми. При обновлении фильтра каждый узел показывается, если его статус входит в список видимых статусов, и скрывается, если не входит. Вместе со скрытием узла скрываются соответствующие линии.

В текущей сцене видимым статусом задан `ACTUAL`.

## 10. Сохранение, загрузка, экспорт и импорт

### 10.1. Локальное сохранение

Комбинация `Ctrl + S` или `Command + S` сохраняет текущую карту в локальное хранилище приложения и одновременно выводит текст сохранения в поле сохранения.

### 10.2. Автозагрузка

При запуске приложение читает последнее сохранение из локального хранилища, помещает его в поле сохранения и пытается загрузить карту.

Если строка сохранения отсутствует или короче 5 символов, загрузка не выполняется.

### 10.3. Ручная загрузка

Карта может быть загружена из текстового поля сохранения:

* кнопкой «Load custom save»;
* комбинацией `Ctrl + O` или `Command + O`.

Перед загрузкой все текущие узлы удаляются, затем список узлов создается заново по данным сохранения.

### 10.4. Экспорт

Комбинация `Ctrl + E` или `Command + E`, а также кнопка «Export save», записывает текущий текстовый код карты в поле сохранения без обязательной записи в локальное хранилище.

### 10.5. Очистка

Комбинация `Ctrl + X` или `Command + X`, а также кнопка «Clear», вызывает функцию очистки, но в текущей реализации она сразу завершает работу и фактически ничего не очищает.

## 11. Формат сохранения

Сохранение — это текст, где каждый узел записан отдельной строкой.

Строка узла состоит из полей, разделенных точкой с запятой:

```text
id;title;relationIds;x|y;size;status;r;g;b
```

Где:

* `id` — идентификатор узла;
* `title` — заголовок узла;
* `relationIds` — идентификаторы связанных узлов, разделенные вертикальной чертой `|`;
* `x|y` — координаты узла;
* `size` — размер узла;
* `status` — числовой статус: `0` актуальный, `1` завершенный, `2` неактуальный;
* `r`, `g`, `b` — компоненты цвета от 0 до 1.

При загрузке:

1. сначала создаются все актуальные узлы с их идентификаторами, заголовками, позициями, размерами, статусами и цветами;
2. затем вторым проходом восстанавливаются связи между созданными узлами;
3. дублирующиеся связи не создаются;
4. связи на отсутствующие или пропущенные узлы игнорируются;
5. после восстановления обновляется видимость узлов и цвета линий.

## 12. Интерфейс

В интерфейсе должны присутствовать:

* рабочая область с узлами и линиями;
* поле редактирования заголовка выбранного узла;
* поле для текстового кода сохранения;
* кнопка создания узла;
* кнопка сохранения текущей карты;
* кнопка загрузки пользовательского сохранения;
* кнопка экспорта сохранения в текстовое поле;
* кнопка очистки, даже если ее текущее действие фактически отключено;
* список кнопок связей выбранного узла;
* краткая текстовая подсказка по управлению;
* три регулятора RGB-цвета выбранного узла.

## 13. Горячие клавиши и действия мыши

| Действие | Управление |
|---|---|
| Переместить вид | удерживать левую кнопку мыши и двигать мышь вне перетаскивания узла |
| Масштабировать вид | колесо мыши без `Shift` |
| Создать новый связанный узел | `Tab` |
| Сохранить заголовок выбранного узла | `Enter` |
| Удалить выбранный узел | `Delete` или `Right Command + Backspace` |
| Создать связь | выбрать узел, затем `Left Shift` + клик по другому узлу |
| Удалить связь | выбрать узел, затем `Left Alt` + клик по связанному узлу |
| Изменить размер выбранного узла | курсор над выбранным узлом + `Left Shift` + колесо мыши |
| Сфокусироваться на выбранном узле | `Ctrl + F` |
| Сохранить в локальное хранилище | `Ctrl/Command + S` |
| Загрузить из поля сохранения | `Ctrl/Command + O` |
| Экспортировать код сохранения в поле | `Ctrl/Command + E` |
| Вызвать очистку | `Ctrl/Command + X`, но фактическая очистка отключена |

## 14. Важные особенности текущего поведения

* Связь между двумя узлами фактически представлена двумя направленными объектами связи, поэтому при визуализации возможно наложение двух линий друг на друга.
* Узлы с неактуальным статусом сохраняются, но при последующей загрузке пропускаются.
* Текстовый формат сохранения не экранирует разделители `;`, `|` и перенос строки, поэтому такие символы в заголовке могут нарушить восстановление данных.
* Функция очистки существует, но отключена ранним завершением.
* Текст внутри узла скрывается, когда узел находится вне видимой области, и снова показывается при возврате в видимую область.
